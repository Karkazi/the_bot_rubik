### 6.7a Синхронизация регистрации Telegram и MAX: как узнать пользователя в другом канале

Цель: один и тот же человек, зарегистрировавшийся в одном канале (Telegram или MAX), при первом входе в другой канал должен быть распознан и привязан к **одному профилю** (общие «Мои заявки», один набор данных). Ниже — правила привязки по каналам.

#### Единый идентификатор в Core

- В Core храним **профиль** по внутреннему **support_user_id** (UUID или автоинкремент). В профиле: `support_user_id`, `telegram_user_id` (опционально), `max_user_id` (опционально), плюс бизнес-поля (ФИО, телефон, email, логин, department и т.д.).
- **Индексы:** по `telegram_user_id`, по `max_user_id`, по `phone` (нормализованный), по `email`, по `login`. Это нужно для поиска профиля при привязке из другого канала.

#### Сценарий 1: Регистрация в MAX → вход в Telegram

1. Пользователь открывает бота (WMS/Lupa/Rubik) в Telegram. Core ищет профиль по `telegram_user_id`. **Не найден** → «первый вход в Telegram».
2. Показываем выбор: **«Привязать аккаунт»** / **«Зарегистрироваться»**.
3. **«Привязать аккаунт»:** запрашиваем **только телефон** — кнопка «Поделиться контактом» (ввод номера с клавиатуры не принимаем). Core ищет профиль по `phone`. Если найден профиль с `max_user_id` и без `telegram_user_id` → дописываем в этот профиль `telegram_user_id`. Сообщение: *«Аккаунт привязан.»* Дальше — общий профиль и общие заявки.
4. Если по телефону профиль не найден — сообщение *«Аккаунт не найден. Зарегистрируйтесь или проверьте номер.»* и снова выбор: привязать / зарегистрироваться.
5. **«Зарегистрироваться»** — полная регистрация (6 шагов). На шаге «телефон» при дубликате номера предлагаем привязать этот Telegram к существующему аккаунту.

#### Сценарий 2: Регистрация в Telegram → вход в MAX

1. Пользователь открывает бота поддержки в MAX. Core ищет профиль по `max_user_id`. **Не найден** → «первый вход в MAX».
2. **Привязать:** только **одноразовый код из Telegram**. В MAX показываем: *«Введите 6-значный код из Telegram-бота поддержки»*. Пользователь в Telegram нажимает «Привязать MAX» → бот выдаёт код (6 цифр), действителен 2–5 минут; код сохраняем в кэш: `code → (support_user_id, telegram_user_id)`. Пользователь вводит код в MAX → Core по коду находит профиль и дописывает `max_user_id`. Email и телефон в MAX не запрашиваем.
3. Если пользователь выбирает «Зарегистрироваться» в MAX — запускаем регистрацию; при совпадении телефона/email с существующим профилем предлагаем привязку.

#### Реестр привязок и уведомления

- «Мои заявки» и уведомления строим по `support_user_id` (все привязанные channel_user_id — telegram и/или max). Политику доставки уведомлений (в оба канала или только канал создания) задаём в конфиге.

#### Краткая сводка

| Ситуация | Как привязать |
|----------|----------------|
| Зарегистрирован в MAX → зашёл в Telegram | **Телефон** — только «Поделиться контактом»; поиск профиля по phone; если найден с max_user_id и без telegram_user_id → дописать telegram_user_id. |
| Зарегистрирован в Telegram → зашёл в MAX | **Одноразовый код из Telegram** (кнопка «Привязать MAX» → 6-значный код, 2–5 мин) → ввод кода в MAX → по коду находим профиль и дописываем max_user_id. Без ввода email/телефона в MAX. |
