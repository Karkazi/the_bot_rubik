# Бот Rubik

Единый бот для **Telegram** и **MAX**: регистрация, заявки в ТП (сайт/Лупа, WMS, смена пароля), личный кабинет, уведомления по заявкам. Админы могут удалять пользователей и получать Excel-отчёт по заявкам Лупа.

## Возможности

- **Регистрация** (в Telegram): ФИО, рабочий логин, корпоративная почта (@petrovich.ru / .tech), телефон. Проверка дубликатов по логину, почте, табельному номеру.
- **Привязать аккаунт**: по номеру телефона — привязка MAX к существующему профилю (или наоборот).
- **Создать заявку в ТП**:
  - **Сайт (Лупа)** — поиск на petrovich.ru: сервис, тип запроса, город, описание; создание заявки в Jira WHD и запись в отчёт.
  - **WMS** — заявки по настройке/проблемам WMS, настройки системы, пользователь PSIwms (с вложениями).
  - **Смена пароля** — создание задачи в Jira AA (логин и телефон из профиля).
- **Мои заявки**: список заявок пользователя со ссылками на Jira (без закрытых/отклонённых).
- **Личный кабинет**: просмотр и изменение учётных данных (ФИО, логин, почта, телефон, подразделения, табельный номер).
- **Уведомления**: при смене статуса заявки и новых комментариях (доставка в Telegram и MAX). Статус «Waiting for customer» не присылается.
- **Админ-панель** (для полных админов и админов Лупы):
  - **Удалить пользователя** (только полные админы): список по 10 на страницу, поиск по ФИО или ввод логина/ID — в Telegram и в MAX.
  - **Отчёт Лупа (Excel)** — скачать файл с заявками по поиску на сайте (доступно полным админам и админам Лупы).

## Роли

- **Полные админы** (`ADMIN_IDS` в Telegram, `ADMIN_MAX_IDS` в MAX): админ-панель, удаление пользователей, отчёт Лупа.
- **Админы Лупы** (`ADMIN_LUPA_IDS` / `ADMIN_LUPA_MAX_IDS`): доступ в админ-панель только к кнопке «Отчёт Лупа (Excel)»; удалять пользователей не могут.

## Архитектура проекта

Один процесс (`main.py`) поднимает Telegram-бота и при наличии токена — бота в MAX. Вся бизнес-логика в **core**; каналы только вызывают API и отображают ответы.

### Структура каталогов

```
the_bot_rubik/
├── main.py                 # Точка входа: Telegram polling + MAX polling + фоновые циклы уведомлений
├── config.py               # Конфигурация из .env (Jira, каналы, админы)
├── user_storage.py         # Профили пользователей и индексы (логин, email, телефон, привязка MAX↔TG)
├── validators.py           # Валидация ФИО, почты, телефона, логина
├── states.py               # FSM-состояния (регистрация, смена данных, админ, WMS, Lupa)
├── keyboards.py            # Клавиатуры Telegram (меню, кнопки)
│
├── core/                   # Бизнес-логика (не зависит от Telegram/MAX)
│   ├── support/
│   │   ├── api.py          # Единый API: get_start, get_main_menu, get_admin_panel, create_ticket, get_my_tickets, delivery
│   │   ├── models.py       # Menu, MenuButton, Text, Error
│   │   ├── ticket_catalog.py  # Каталог типов заявок (YAML)
│   │   ├── issue_binding_registry.py  # Реестр привязок заявка → (channel, user_id)
│   │   └── delivery.py     # Доставка сообщений в Telegram и MAX по (channel_id, user_id)
│   ├── registration.py     # Регистрация и обновление учётных данных
│   ├── password.py         # Запрос смены пароля (Jira AA)
│   ├── password_requests.py # Ожидающие заявки на смену пароля, опрос статусов
│   ├── jira_aa.py          # Jira AA: создание задачи, статус, комментарии, get_issue_info
│   ├── jira_wms.py         # Jira PW (WMS): создание заявок (ошибка, настройки, PSIwms), вложения
│   ├── jira_wms_departments.py  # Справочник подразделений WMS из Jira
│   ├── jira_lupa.py        # Jira WHD (Лупа): создание заявки по поиску
│   ├── jira_departments.py # Справочник подразделений AA
│   ├── wms_constants.py    # Процессы и типы услуг WMS
│   ├── notifications.py    # Опрос статусов/комментариев по реестру, отправка уведомлений
│   └── lupa_report.py      # Excel-отчёт по заявкам Лупа (data/lupa_tickets_log.xlsx)
│
├── handlers/               # Обработчики Telegram (вызов core + рендер в TG)
│   ├── start.py            # /start, привязка аккаунта, back_to_main
│   ├── registration.py    # Регистрация (FSM)
│   ├── credentials.py     # Изменение учётных данных (FSM)
│   ├── password.py        # Смена пароля
│   ├── create_ticket.py   # Создать заявку: раздел Сайт/WMS/Пароль, сценарии WMS и Lupa
│   ├── my_tickets.py      # Мои заявки, просмотр заявки, комментарии
│   ├── comments.py        # Добавить комментарий к заявке
│   ├── admin.py           # Админ-панель, удаление пользователя, отчёт Лупа
│   └── menu_extra.py      # Личный кабинет, помощь
│
├── adapters/
│   ├── telegram/          # Рендер ответов core → формат Telegram (render.py)
│   └── max/               # Адаптер MAX
│       ├── main_max.py    # Long polling MAX, маршрутизация callback/сообщений
│       ├── handlers.py    # Обработка callback (меню, админка, удаление пользователя)
│       ├── render.py      # Core Menu/Text/Error → ответ MAX
│       ├── wms_flow.py    # Сценарий создания заявки WMS в MAX
│       └── lupa_flow.py   # Сценарий создания заявки Лупа в MAX
│
├── middlewares/            # Middleware (антиспам и др.)
├── config/                # YAML-конфиги (ticket_catalog.yaml)
├── data/                  # JSON-хранилище (профили, реестр привязок, состояние уведомлений, кэши)
└── scripts/               # Вспомогательные скрипты (Jira request type, option ids и т.д.)
```

### Поток данных

- **Пользователь** (Telegram или MAX) нажимает кнопку или отправляет текст.
- **Адаптер** (handlers + adapters/telegram или adapters/max) определяет действие, вызывает **core.support.api** (get_main_menu, create_ticket, get_my_tickets и т.д.).
- **Core** обращается к **user_storage**, **issue_binding_registry**, **config**, к модулям Jira (jira_aa, jira_wms, jira_lupa) и возвращает **Menu** / **Text** / **Error** или результат создания заявки.
- **Адаптер** превращает ответ в сообщение и кнопки канала и отправляет пользователю.
- **Уведомления**: фоновые задачи в `main.py` вызывают `core.notifications` (опрос статусов и комментариев по реестру); **core.support.delivery** шлёт сообщения в Telegram и MAX.

### Ключевые зависимости слоёв

| Слой        | Зависит от |
|------------|------------|
| handlers/  | core.support.api, user_storage, config, keyboards, states |
| adapters/max/ | core.support.api, user_storage, config, core.* (jira_wms, jira_lupa) |
| core/      | config, user_storage, issue_binding_registry, Jira-модули |
| user_storage | только config (пути), файловая система |

## Установка и запуск

1. **Требования**: Python 3.10 или новее.

2. **Клонирование или копирование проекта** в нужную папку.

3. **Виртуальное окружение** (рекомендуется):
   ```bash
   python -m venv venv
   # Windows:
   venv\Scripts\activate
   # Linux/macOS:
   source venv/bin/activate
   ```

4. **Установка зависимостей**:
   ```bash
   pip install -r requirements.txt
   ```

5. **Настройка окружения**: скопировать `.env.example` в `.env` и заполнить переменные (см. раздел «Настройка» ниже).

6. **Запуск**:
   ```bash
   python main.py
   ```
   Запускаются Telegram-бот и при наличии `MAX_BOT_TOKEN` — бот в MAX в одном процессе. Логи выводятся в консоль; при настройке логирования — также в `data/bot.log`.

Остановка: `Ctrl+C` в терминале.

## Настройка

1. Скопировать `.env.example` в `.env`.
2. **Telegram**: `TELEGRAM_TOKEN`, `ADMIN_IDS` (ID через запятую). Опционально: `ADMIN_LUPA_IDS`.
3. **MAX**: при необходимости — `MAX_BOT_TOKEN`, `ADMIN_MAX_IDS`, `ADMIN_LUPA_MAX_IDS`.
4. **Jira**: `JIRA_LOGIN_URL`, `JIRA_TOKEN`. Для проекта AA (смена пароля) — поля и при необходимости Service Desk ID/Request Type. Для WHD (Лупа) и PW (WMS) — свои URL и настройки в конфиге.
5. Уведомления и опрос статусов/комментариев запускаются из `main.py` (интервалы через переменные окружения при необходимости).

## Проверка

1. В `.env` заданы минимум `TELEGRAM_TOKEN` и `ADMIN_IDS`.
2. В корне: `python main.py` — в консоли сообщение о запуске.
3. В Telegram: `/start` → главное меню; «Создать заявку в ТП» → выбор Сайт / WMS / Смена пароля; «Личный кабинет», «Мои заявки», «Помощь». При вашем ID в `ADMIN_IDS` — «Админ-панель» с удалением пользователя и отчётом Лупа.
4. В MAX: тот же сценарий (меню из core); удаление пользователя — выбор способа (список / поиск по ФИО / логин или ID), затем выбор пользователя и подтверждение.
5. Свой Telegram ID можно узнать, например, у [@userinfobot](https://t.me/userinfobot).

## Зависимости
