# Безопасность данных бота Rubik

## 3. Может ли сторонний человек получить информацию о пользователях?

**При текущей реализации:**

- Данные хранятся в файлах на сервере (`data/user_data.json`, индексы в `data/`). Доступ к ним имеет только тот, у кого есть доступ к файловой системе сервера (администратор, хостинг).
- Публичного API для чтения данных нет: бот не отдаёт списки пользователей или персональные данные наружу.
- Telegram-бот не может «узнать» пользователей, которые не написали боту сами; идентификация только по `user_id` в чате с ботом.

**Вывод:** Сторонний человек не может получить данные пользователей через бота или через интернет. Риск только при компрометации сервера (доступ к файлам, бэкапам, логам) или утечке токена бота (тогда злоумышленник может работать от имени бота, но не читать JSON-файлы напрямую).

**Рекомендации:** ограничить доступ к папке `data/` и к `.env` (права доступа, отдельный пользователь для процесса бота), не класть секреты в репозиторий, хранить бэкапы в защищённом месте.

## 2. Можно ли зашифровать данные, чтобы их нельзя было использовать в незаконных целях?

**Да.** Добавлено опциональное шифрование персональных данных «в покое»:

- Включается переменной окружения `ENCRYPT_USER_DATA=1` и ключом `USER_DATA_ENCRYPTION_KEY` (строка в base64, сгенерированная через `cryptography.fernet`).
- Шифруются поля: ФИО, логин, почта, телефон. В файлах хранятся только зашифрованные значения; без ключа прочитать их нельзя.
- Это защищает от несанкционированного использования данных при утечке файлов или бэкапов (без ключа данные не пригодны для использования).

**Как включить:** в `.env` задать:
- `ENCRYPT_USER_DATA=1`
- `USER_DATA_ENCRYPTION_KEY=<ключ в base64>`

Ключ сгенерировать один раз (Python):
```python
from cryptography.fernet import Fernet
print(Fernet.generate_key().decode())
```
Значение поместить в `USER_DATA_ENCRYPTION_KEY`. **Важно:** при смене ключа старые данные не расшифровать; делайте бэкап и храните ключ в надёжном месте.

|--------|--------|
